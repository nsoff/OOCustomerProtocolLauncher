# OOCustomerProtocolLauncher


---

## 介绍

**OOCustomerProtocolLauncher** 是一个使用 **.NET 10** 开发的 **轻量级**、**开源** 的实用工具。它旨在解决一个常见且棘手的问题：如何让一个简单的网页链接或外部指令，直接调用用户本地已安装的 **Microsoft Office** 应用程序（如 Excel, Word, PowerPoint）来打开指定的 **网络文件** 或 **本地文件**。

传统上，点击网页上的标准链接（`http://` 或 `https://`）通常会在浏览器中打开或尝试进行在线预览，而无法直接启动本地的应用程序。本程序通过注册并监听一个自定义的 URL 协议（例如 `myoffice://`），巧妙地绕过了此限制，在网页与本地应用之间架起了一座桥梁，为用户提供了无缝、流畅的本地应用启动体验。

---

## 核心功能

- **一键启动本地 Office**：安装本程序后，用户只需点击网页或邮件中的特定链接，即可立即在本地 Office 应用中打开目标文件。
- **自定义 URL 协议**：程序会注册一个易于识别的自定义协议（默认为 `myoffice://`）。您可以轻松地构建链接，以指定要打开的文件路径或 URL。
- **支持网络与本地文件**：无论是托管在云端的 `.xlsx` 文件，还是本地硬盘上的 `.docx` 文档，都可以通过链接直接打开。
- **提升工作效率**：为需要频繁从网页或系统中访问特定 Office 文件的工作流程提供便利。

## 快速开始与使用示例

### 1. 安装

使用publish.bat编译发布文件,利用setup.iss脚本生成安装包进行安装。

### 2. 使用

安装成功后，您就可以开始创建和使用自定义链接了。

**语法**: `myoffice://open?file=<文件URL或本地路径>`

**示例**: 在网页的 `<a>` 标签中，您可以这样写：
```html
<a href="myoffice://open?file=https://www.sample.com/a.xlsx">点击此处，使用本地 Excel 打开 a.xlsx</a>
